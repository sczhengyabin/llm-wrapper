# LLM Wrapper - WebUI 增强与协议管理

## 项目背景

llm-wrapper 是一个轻量级的 OpenAI/Anthropic 协议聚合网关，为多个上游 LLM 服务提供统一入口。当前版本已实现核心的配置管理、模型路由、参数注入和调试功能。

## 项目目标

增强 WebUI 的模型管理能力，添加协议级别的控制，改善用户体验。

## 需求概述

### 1. 上游模型快速添加 Alias
- 上游列表中显示的模型名字改为可点击
- 灰色框 → 可点击交互
- 启用状态：绿色（启用）/ 红色（不启用/重名）
- 添加上游后默认启用（除非重名）
- 点击模型名直接创建同名透传 alias

### 2. Alias 标签系统
- 添加 tag 标记：`auto` / `manual`
- `auto`: 跟随上游自动创建的 alias
- `manual`: 手动添加的 alias

### 3. 重名检测与处理
- 自动和手动添加 alias 时检查重名
- **Auto 情况**:
  - 重名时默认不添加，显示红色
  - 用户修改手动 alias 名字不重名后，可再点击变为绿色添加成功
- **Manual 情况**:
  - 重名则添加失败，提醒用户

### 4. 协议支持开关
- 在上游编辑中，为 OpenAI 和 Anthropic 协议分别设置支持开关
- 配置示例：
  ```yaml
  upstream:
    name: anthropic
    protocols:
      openai: false
      anthropic: true
  ```
- 客户端请求不支持的协议时，直接返回 422，不转发给上游

### 5. WebUI 侧边栏显示模型列表
- 页面顶端/侧边栏显示当前服务聚合的 v1/models 返回的模型列表
- 采用侧边栏形式展示

## 技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| Alias 创建方式 | 点击直接创建同名 | 简化操作，符合透传场景 |
| 协议开关粒度 | 上游级别 | 更符合上游能力的实际场景 |
| 模型列表展示 | 侧边栏 | 更好的可视性和导航体验 |

## 范围边界

**包含**:
- WebUI 交互增强
- 上游配置模型扩展
- Alias 标签系统
- 协议路由逻辑

**不包含**:
- 后端 API 结构大改
- 新的协议支持
- 性能优化

## 成功标准

1. 用户可点击上游模型名快速创建 alias
2. Alias 有明确的 auto/manual 标签
3. 重名情况有清晰的视觉反馈和处理流程
4. 协议开关可控制上游支持的协议
5. 侧边栏显示所有可用模型
